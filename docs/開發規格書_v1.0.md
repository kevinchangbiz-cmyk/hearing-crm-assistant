# Claude Code 最終開發規格書 v1.0 — Hearing CRM Assistant

## 專案背景
助聽器門市驗配師每日需花費大量時間撰寫 CRM 服務紀錄、補登資訊及建立追蹤提醒。本系統目標為將「服務完成 → 完成紀錄」縮短至 30 秒內。

## 產品定位
這是一套助聽器門市專用行政助手，不是 CRM、ERP、Agent 平台。核心任務只有：語音/文字輸入 → AI 整理 → CRM 輸出 → 備份 → 提醒。

## 核心使用流程
1. Google 登入
2. 選擇店別
3. 輸入客戶姓名
4. 語音或文字輸入
5. Gemini 整理 CRM
6. 顯示 CRM 結果
7. 一鍵複製 CRM
8. 自動備份 Google Sheet
9. 如有追蹤事項，可建立 Calendar 提醒

## 技術架構 (原始規格)
- Frontend：Next.js
- Backend：Supabase *(已移除，見 README.md)*
- AI：Gemini 2.5 Flash
- Authentication：Google OAuth
- Storage：Supabase *(改為 Google Sheets)*
- Integration：Google Sheets API、Google Calendar API
- Deployment：Vercel

## 資料模型 (原始規格)
- **Users**：id, email, name
- **Stores**：id, store_name, google_sheet_id, google_calendar_id
- **User_Store_Permissions**：user_id, store_id
- **Records**：id, store_id, customer_name, raw_text, crm_result, created_by, created_at

> 註：依最新決策，Users / Stores / Permissions 改以 `config/stores.json` 管理；Records 改寫入 Google Sheet。

## 店別權限邏輯
使用者登入後只能看到被授權店別。範例：
- 小K → 員林店、彰化中華店
- 阿明 → 彰化中華店

## 首頁介面
- 區塊1：店別選擇
- 區塊2：客戶姓名
- 區塊3：錄音按鈕
- 區塊4：文字輸入框
- 區塊5：產生 CRM 按鈕
- 區塊6：CRM 結果區
- 區塊7：建立 Calendar 提醒
- 區塊8：最近 20 筆紀錄 *(降為非必要)*

## CRM 六大分類
保養、調整、維修、聽檢、總公司客服、其他。

## CRM 格式規則
輸出格式：

```
<保養>內容
<調整>主訴 調整內容 結果
<維修>內容
<聽檢>內容
<總公司客服>內容
<其他>內容
```

無內容分類不得顯示。

## 助聽器專業判斷規則
- **保養**：清潔、抽濕、更換耳塞、耳管、濾網
- **調整**：音量、清晰度、電話、電視、方向性等調整
- **維修**：故障、無聲、受潮、送修
- **聽檢**：純音、語音、新客試聽、追蹤聽檢
- **其他**：購買資訊、贈品、付款方式、追蹤事項

## Google Sheet 規格
每店一份 Sheet。欄位：建立時間、店別、客戶姓名、原始內容、CRM 內容、建立者。用途：備份，不做統計。

## Google Calendar 規格
AI 偵測以下內容：三個月後追蹤、下週回診、指定日期、聯絡提醒。
**建立事件前需顯示確認按鈕。**
標題規則：
- 一般客戶：姓名 + 服務類型
- 新客：姓名 + 新客試聽

## 語音輸入規格
支援桌機與手機。按下錄音後開始收音。停止錄音後送至 Gemini。轉錄文字需顯示供使用者修改。

## Gemini Prompt 核心要求
你是台灣助聽器驗配師行政助手。請將內容整理為 CRM 格式。保持專業、簡潔。避免流水帳。自動分類六大項。保留所有重要技術調整內容。若有追蹤需求請另外輸出 Reminder 欄位。

## 最近紀錄功能
顯示最近 20 筆。可檢視：原始內容、CRM 內容、建立時間、店別。*(降為非必要功能)*

## 手機版需求
RWD。單手操作。錄音按鈕置中放大。

## V1 不開發
LINE Bot、客流量統計、銷售統計、庫存管理、ERP 整合、CRM 自動登入、CRM 自動填寫、Agent、Hermes、AutoGPT。

## 驗收標準
- 30 秒內完成一筆服務紀錄
- CRM 格式符合門市習慣
- Google Sheet 正常寫入
- Google Calendar 正常建立
- 手機與桌機皆可使用
