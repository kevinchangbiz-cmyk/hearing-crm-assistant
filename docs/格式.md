# CRM 內容格式

> 僅針對內容，而不是只 CRM 服務紀錄選單的六大類。

## 一般格式

```
<保養> 輸入內容
<調整> 輸入內容
<聽檢> 輸入內容
<維修> 輸入內容
<其他> 輸入內容
```

## 新客聽檢特殊格式

唯獨六大類的「聽檢」比較特殊。新客一定是選「聽檢」，格式與定義如下。因內容會有重疊很難定義，不要太離譜就好。

```
<病史與行為觀察>
聽損多久、有無相關病史 (中耳炎、耳中風等聽力相關的病史) 或其他慢性病、有無就醫、
醫生診斷為何、有無試聽過助聽器、以前有佩戴助聽器的經驗嗎、是否有病識感、誰陪同前來

<個案期待與需求>
覺得困擾的是甚麼、動機、想改善的部分、預算、等級、價位、功能等、有無去過好市多

<耳科評估>
L:
R:
記錄檢耳情況：乾耳/濕耳/油耳、聽損程度、感音神經型/傳導型/混合型聽損、語辨度 N%

<試聽狀況>
試聽機型順序 1>2>3
1. 機型1
   - 報價、優惠價方案 (EX. 父親節方案 88000)
   - 試聽情形 (EX. 一開始覺得吵、覺得聽不清楚...諸如此類)
2. 機型2：同上
最後選擇某機型、原因等等

<計畫與建議>
想借回家試聽、建議多久追蹤、建議考慮換機

<備註>
購:購買的機型*數量
贈:充電盒、電池、耳模等
```
